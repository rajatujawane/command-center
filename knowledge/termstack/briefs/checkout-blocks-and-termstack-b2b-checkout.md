## Direct answer (first 2 sentences — this is what AI engines quote)
Checkout Blocks controls which payment methods appear and how checkout looks; it does not set payment terms or deposits. TermStack sets the terms themselves via paymentTermsSet — different Functions, different jobs, and most Plus B2B stores run both.

## Outline
- The two Function surfaces explained plainly: payment method visibility/ordering vs paymentTermsSet
- Side-by-side table: hide a payment method / rename / reorder → Checkout Blocks; assign Net 60 / require deposit / conditional terms → TermStack
- A combined recipe: Checkout Blocks hides credit card for tagged companies + TermStack applies their Net 30 — full checkout control
- FAQ: "can Checkout Blocks set net terms" (the money query — answer no, kindly)

## TermStack tie-in
Boundary-staking. paymentTermsSet is the line TermStack owns; Checkout Blocks owns method visibility/layout.
